# .impeccable.md

이 프로젝트의 디자인 결정 기준. `/impeccable` 계열 스킬과 디자인 작업 전에 먼저 읽는다.

## Design Context

### Users

**더 프롬프트 컴퍼니(TPC)의 교육·데모 자료를 보는 사람.** 직접 매일 쓰는 소비자 제품이 아니라, 강의나 세미나 자리에서 화면에 띄워 설명하기 위한 **포트폴리오성 데모**다. 실제 관객은:

- LLM/에이전트를 배우러 온 수강생·실무자
- 회사 소개 자리의 의사결정자

그래서 평가 기준은 "오래 쓰기 편한가"보다 **"보자마자 완성도 있어 보이는가, 모든 디테일이 살아 있는가"**. 빈 상태·로딩·다크 모드·키보드 네비 같은 부분이 다 챙겨져야 데모 가치가 산다.

### Brand Personality

세 단어: **차분한 / 신뢰감 / 전통적 (신문사스러운)**.

- 보이스: 차분하고 단정한 신문 사설 톤. "AI가 정리해줍니다!" 같은 마케팅 어조 금지.
- 감정 목표: 차분한 권위(quiet authority). 정보가 정돈돼 있다는 안도감.
- 한국 정통 저널리즘(NYT·FT·한겨레·중앙 등의 모바일/웹) 정서. 다만 그대로 베끼는 게 아니라 **에디터리얼 무드만 차용**한다.

### Aesthetic Direction

- **방향**: Editorial / newsprint. 종이 같은 따뜻한 톤. 활자 위주, 장식 최소.
- **이론**: 흰 종이 같은 오프화이트(라이트) + 따뜻한 near-black(다크). 와인/딥레드 액센트 1색만, 절제해서 사용.
- **테마**: OS `prefers-color-scheme` 자동 추종. 토글 없음. 라이트/다크 양쪽 동등하게 완성한다.
- **타이포**: 한국어 본문 Gowun Dodum + 디스플레이 Nanum Myeongjo. 모노는 Geist Mono(로그/코드 박스 전용).

**Anti-references (이렇게는 안 됨)**:
- ❌ B2B SaaS / Vercel·Linear·Stripe 포트폴리오 풍의 보라색 그라데이션 + 글래스모피즘 + 깊은 소프트 그림자
- ❌ 한국 포털 뉴스(네이버·다음) 같은 정보 밀도 폭주 + 광고 + 회색 박스 정렬
- ❌ 화려한 랜딩페이지 / 하이퍼 애니메이션 (parallax, scroll-jacking, 빵빵 터지는 마이크로 인터랙션)
- ❌ 일반 AI 챗봇 UI 클리셰 (네온 다크 + 보라/시안 + 활자 윈도우)

### Design Principles

이후 어떤 디자인 결정이든 다음 다섯 원칙으로 검증한다.

1. **신문 같은 안정감을 우선한다.**
   움직임은 페이지 진입 시점의 절제된 페이드/이즈아웃 한 번 정도면 충분. 호버 시 카드가 들썩이거나 버튼이 통통 튀는 식의 마이크로 인터랙션은 신문 정서와 충돌한다.

2. **읽기 편안함이 시각적 매력보다 우선한다.**
   본문 줄높이 1.6 이상, 본문 너비 65–75ch, 한국어 가독을 위한 letter-spacing 미세 조정. 멋있어 보이는 좁은 컬럼/얇은 글자 두께로 가독성을 깎지 않는다.

3. **장식 대신 위계로 말한다.**
   대비는 크기·굵기·간격·여백으로. 그라데이션·소프트 그림자·illustrative 아이콘은 아껴 쓴다. 와인 액센트 한 색만 의미 있는 곳(primary action, 핵심 강조)에 절제해서 사용.

4. **데모인 만큼 모든 상태가 챙겨져 있어야 한다.**
   빈 상태(empty), 로딩, 에러, 다크 모드, 키보드 포커스, 줄어든 모션(`prefers-reduced-motion`), 모바일 반응형, WCAG AA 콘트라스트 — 발표 중에 우연히 깨지는 게 없도록 모든 상태가 디자인의 일부다.

5. **편향 없이 토큰 시스템 안에서 결정한다.**
   하드코딩 색·픽셀 절대 없음. 모든 색은 OKLCH 토큰, 모든 간격은 4pt 스케일 토큰. 라이트/다크는 토큰 값의 두 세트로 정의되며 컴포넌트 코드는 양쪽에서 동일하게 동작해야 한다.

### Token Anchors

현재 `src/app/globals.css`에 정의된 시스템:

- **컬러**: `--surface`, `--surface-2`, `--surface-sunken`, `--surface-code`, `--text`, `--text-strong`, `--text-muted`, `--border`, `--border-strong`, `--accent`(wine), `--accent-soft`, `--info`/`--success`/`--warning`/`--danger`
- **간격**: `--space-2xs` … `--space-3xl` (4pt scale)
- **라운드**: `--radius-sm|md|lg|pill`
- **타입 스케일**: `--text-xs` … `--text-3xl`
- **폰트**: `--font-display`(Nanum Myeongjo), `--font-body`(Gowun Dodum), `--font-mono`(Geist Mono)
- **컨테이너**: `--content-narrow`(760), `--content-wide`(960)

### Accessibility / Inclusion

- WCAG **AA** (4.5:1) 최소 보장. 가능한 곳은 AAA.
- 모든 인터랙티브 요소 최소 44×44 px 터치 타깃.
- 키보드 네비 전 화면 가능, `:focus-visible` 가시.
- `prefers-reduced-motion` 존중 — 모션은 거의 0에 가깝게.
- `lang="ko"`, 한국어 스크린리더 친화 라벨링.
